Network Threats and Attacks

Understanding Network Defenses

Types of Threats

Spoofing

An attack that uses a falsified identity to gain access to a system.

Phishing

An attack that tricks users into visiting malicious websites, often through fake emails or links.

DoS/DDoS

A denial-of-service attack that prevents legitimate users from accessing a system. DDoS attacks use multiple systems to launch the attack.

Malware

A program intended to compromise the confidentiality, integrity, or availability of a system.

Malware and Other Threats

Virus

A self-replicating code that spreads with the user's assistance (e.g., clicking a link).

Worm

A self-replicating code that spreads without any human intervention.

Trojan

A seemingly harmless program that carries a malicious payload, like ransomware, which encrypts files for a ransom.

On-path (MITM) Attack

An attack where the attacker places themselves between two devices to intercept or modify communication.

Advanced and Insider Threats

Side-channel

A passive, non-invasive attack that observes a device's operation (e.g., by monitoring power consumption).

Advanced Persistent Threat (APT)

A highly sophisticated and long-term attack carried out by organized groups.

Insider Threat

A threat that originates from individuals who are trusted by the organization, such as disgruntled employees or victims of a scam.

Identifying and Preventing Threats

Various tools can be used to protect networks. Some can only identify threats, while others can prevent them from happening. It's always best practice to not run unneeded services to reduce the attack surface.

  • Identify Threats Only: IDS, HIDS, NIDS, SIEM, Scans
  • Identify and Prevent Threats: Anti-malware/Antivirus, Firewall, IPS (NIPS/HIPS)

Intrusion Detection System (IDS)

An IDS is a monitoring tool that inspects logs and real-time events to detect abnormal activity, such as intrusion attempts. It is a key part of a defense-in-depth strategy. [Image of IDS diagram]

Host-based IDS (HIDS)

Monitors a single computer. It can examine events in great detail and pinpoint specific files compromised in an attack. It's more costly to manage per system.

Network-based IDS (NIDS)

Monitors and evaluates network activity. A single NIDS can monitor a large network and has little effect on overall performance.

Security Information and Event Management (SIEM)

A SIEM solution collects log data from various sources across an entire enterprise. The goal is to gain a better overall understanding of security concerns and streamline security efforts. It helps to correlate events from different systems to identify potential attacks that might otherwise be missed.

Intrusion Prevention System (IPS)

An IPS is a special type of active IDS that automatically attempts to detect and block attacks before they reach target systems. The key difference is that an IPS is placed in-line with the traffic, allowing it to actively stop malicious packets. This function is often integrated into firewalls.

Firewalls

Firewalls act as a barrier to enforce security policies by filtering network traffic based on a set of rules. They can be network-based, protecting entire networks, or host-based, protecting a single system. Modern firewalls, often called Next-Generation (NG) Firewalls, integrate a variety of threat management capabilities into a single framework.

  • Traditional Firewalls: Filter traffic at Layers 2, 3, and 4 (MAC, IP, and Ports).
  • NG Firewalls: Add filtering based on user identity, antivirus, and application-level control.

Antivirus and Scans

Anti-malware/Antivirus

Identifies malicious software based on known signatures or by detecting abnormal activity. Modern solutions provide a more holistic approach, protecting against viruses, ransomware, and spyware.

Scans

Regular vulnerability and port scans evaluate the effectiveness of security controls. They help identify areas where patches or security settings are insufficient, revealing vulnerabilities before attackers can exploit them.
