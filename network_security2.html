More Networking Terms

Understanding the basics of data communication.

Key Networking Terms

Ethernet (IEEE 802.3)

A standard for wired network connections that defines how data is formatted to ensure devices can communicate over the same cables.

Media Access Control (MAC) Address

A unique physical address assigned to every network device. The first 3 bytes identify the manufacturer. No two devices on the same local network can have the same MAC address.

Internet Protocol (IP) Address

A unique logical address that represents a network interface. It's used for communication and can be reassigned, unlike a MAC address which is tied to the hardware.

Why We Need Networking Models

Networking models provide a structured way to design and understand how different hardware and software systems connect and communicate. They help us break down complex processes into manageable layers.

  • Provide reliable, managed communications between hosts.
  • Isolate functions in different layers.
  • Be vendor-agnostic, scalable, and resilient.

A Simple Two-Layer Model

Upper Layer (Application)

Manages the connection, controls the session, and ensures data is formatted for the receiving application. It's where applications communicate with the network.

Lower Layer (Media/Transport)

Handles the physical connection, converting bits into frames and adding routing data to create packets. It's responsible for getting data ready to be sent.

The Open Systems Interconnection (OSI) Model

The OSI model is a conceptual framework that divides network communication into seven distinct layers. Each layer has specific responsibilities and communicates with the layers directly above and below it. This provides a common reference for understanding how different protocols and hardware work together.

OSI Layers and Their Functions

Application, Presentation, Session (L5-7)

These are the "data" layers. They handle application-specific communication, data formatting, and managing connections. Common terms here include JPEG files (Presentation) and NetBIOS (Session).

Transport, Network, Data Link (L2-4)

These layers are all about moving data. The Transport Layer (L4) handles protocols like TCP and UDP. The Network Layer (L3) manages routing and packets with devices like routers. The Data Link Layer (L2) handles frames and devices like switches.

Physical (L1)

This is the hardware layer, where data is converted into binary (0s and 1s) and sent across physical wires like an Ethernet cable.

Encapsulation and De-encapsulation

Encapsulation is the process of adding headers and trailers as data moves down the OSI model. Each layer's data unit is treated as the payload for the next lower layer, making the total size larger.

De-encapsulation is the reverse process. As data moves up the model, the headers and trailers are removed to properly interpret the payload. The data unit becomes smaller.

The TCP/IP Model

Developed before the OSI model, the TCP/IP protocol stack is the most widely used today. It focuses on the core functions of networking and is simpler with four layers.

TCP/IP Layers

  • Application Layer
  • Transport Layer
  • Internet Layer
  • Network Interface Layer

Corresponding OSI Layers

  • Application, Presentation, Session
  • Transport
  • Network
  • Data Link, Physical

TCP vs. UDP

Transmission Control Protocol (TCP)

A full-duplex connection-oriented protocol. It establishes a connection before sending data, ensuring all packets are received and in the correct order. Great for reliable data transfer like web pages and emails.

User Datagram Protocol (UDP)

A simplex connectionless protocol. It sends data without establishing a connection first. It's faster but less reliable, making it ideal for real-time applications like video streaming and online gaming where speed is more important than every single packet arriving.

Internet Protocol Version 4 (IPv4)

IPv4 uses a 32-bit address space, written as four octets separated by dots (e.g., 192.168.1.1). It's divided into a network number and a host part. Due to a limited number of public addresses, private address ranges were created to be reused within local networks.

Private IPv4 Address Ranges

These addresses can be used by anyone for internal networks, allowing for address reuse without conflicts on the public internet.

  • 10.0.0.0 to 10.255.255.254
  • 172.16.0.0 to 172.31.255.254
  • 192.168.0.0 to 192.168.255.254

The Loopback Address

The address 127.0.0.1 is reserved for a computer's loopback address. It allows a machine to send a network packet to itself, which is useful for troubleshooting and self-diagnosis.

Internet Protocol Version 6 (IPv6)

IPv6 is a modernization of IPv4 that addresses its shortcomings. It uses a 128-bit address space, which provides a virtually unlimited number of addresses. An example address is 2001:db8::ffff:0:1.

  • Larger address field: Eliminates the shortage of public addresses.
  • Improved security: IPsec is a mandatory component.
  • Improved Quality of Service (QoS): Helps services get the bandwidth they need.
