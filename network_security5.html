Network Security Infrastructure

Understanding Core Components and Strategies

On-Premises Data Centers

An on-premises data center is a facility owned and operated by an organization. It requires careful management of several physical aspects to ensure security and reliability.

  • Power: Needs to be constant and consistent.
  • HVAC: Required for cooling high-density equipment.
  • Fire Suppression: Specialized systems (like gas-based) to prevent damage to electronics.
  • Redundancy: Designing systems with duplicate components and diverse power sources to prevent failures.

Cloud Computing

Cloud computing provides on-demand access to a shared pool of configurable computing resources (networks, servers, storage, etc.). It offers several key characteristics:

  • On-Demand Self-Service & Broad Network Access
  • Resource Pooling & Rapid Elasticity
  • Measured Service (pay-per-use)
  • Reduced costs and scalability

Cloud Service Models

These models define the level of control and responsibility between the user and the cloud provider.

Model Description User Responsibility
Software as a Service (SaaS) Access to software applications over the internet (e.g., email, productivity tools). Minimal administration.
Platform as a Service (PaaS) An environment for building and operating your own software (e.g., development tools). Managing applications and some configurations.
Infrastructure as a Service (IaaS) Basic computing resources like servers, storage, and networking. Installing and maintaining operating systems and applications.

Cloud Deployment Models

  • Public Cloud: Shared resource open to the public.
  • Private Cloud: Developed for a single private organization.
  • Hybrid Cloud: Combines a public and private cloud.
  • Community Cloud: Developed for a specific community with shared interests.

Key Agreements & Providers

  • Managed Service Provider (MSP): A company that manages IT assets for another company.
  • Service-Level Agreement (SLA): A legal contract that specifies the minimum level of service, availability, and security a cloud provider must deliver.
  • MOU/MOA: Agreements between organizations to share resources for business continuity and disaster recovery.

Network Design Concepts

  • Network Segmentation: Controlling traffic among networked devices.
  • DMZ: A network area accessible to outside visitors, but isolated from the private network.
  • VLANs: Logically segmenting a network without changing its physical topology.
  • VPNs: A point-to-point communication tunnel over an untrusted network.

Defense in Depth

This is a layered approach to security. The goal is to use multiple types of controls to prevent a single point of failure. Think of a castle with multiple layers of defense.

  • Physical: Locks, walls, access control.
  • Perimeter: Gateway firewalls, secure DMZs.
  • Internal Network: IDS, IPS, internal firewalls.
  • Host: Antivirus, endpoint firewalls.
  • Application: Firewalls, data leak prevention.
  • Data: Encryption, access management.

Zero Trust

Zero Trust is an evolving design approach that assumes a network perimeter is not enough. It adds defenses at the user, asset, and data level, insisting that every process or action a user attempts must be authenticated and authorized. This is often achieved through microsegmentation.

Network Access Control (NAC)

NAC is a concept for controlling access to a network. It ensures that all devices, whether from an employee or a guest, comply with an organization's security policies before they are allowed to connect. Noncompliant devices can be isolated or quarantined.
