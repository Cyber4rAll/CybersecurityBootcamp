Network Security and Protocols

Understanding Ports, Protocols, and Wi-Fi

What is Wi-Fi?

Wi-Fi is a wireless networking technology that uses radio waves to provide high-speed internet and network access to devices like phones, laptops, and smart home devices. It works by sending signals from a wireless router or access point, which devices can pick up to connect to a larger network, such as the internet.

Security of the Network

The TCP/IP protocol stack has numerous vulnerabilities. Improperly implemented stacks can be vulnerable to various attacks like DoS/DDoS attacks, fragment attacks, oversized packet attacks, spoofing attacks, and man-in-the-middle attacks. TCP/IP is also subject to passive attacks through network monitoring or sniffing, which is the act of monitoring traffic to gain information about a network.

Ports and Protocols

Physical Ports

These are the actual, tangible ports on a device like a router, switch, or computer where you plug in a network cable.

Logical Ports

Also called a socket, a logical port is a number that allows a single IP address to support multiple simultaneous communications. Each communication uses a different port number.

Port Categories

  • Well-known ports (0–1023): Used for common, core protocols like HTTP (port 80) and DNS (port 53).
  • Registered ports (1024–49151): Associated with specific proprietary applications and vendors, like Microsoft SQL Server.
  • Dynamic or private ports (49152–65535): These are used for temporary sessions and are released once the communication is over.

Insecure vs. Secure Ports

Some protocols transmit information in clear text, which is unencrypted and vulnerable to network sniffing. Secure alternatives use encryption to protect data in transit. Below are some examples.

Insecure Port Protocol Secure Alternative Protocol
21 FTP 22 SFTP
23 Telnet 22 SSH
25 SMTP 587 SMTP with TLS
80 HTTP 443 HTTPS
143 IMAP 993 IMAP over SSL/TLS
389 LDAP 636 LDAPS

The TCP 3-Way Handshake

TCP is a connection-oriented protocol that establishes a reliable connection before sending data. This is done through a "three-way handshake" consisting of three messages:

  • SYN (Synchronize): The client sends a SYN packet to the server to initiate a connection.
  • SYN-ACK (Synchronize-Acknowledge): The server responds with a SYN-ACK packet, acknowledging the client's request and sending its own synchronization request.
  • ACK (Acknowledge): The client sends an ACK packet to the server, acknowledging the server's request. The connection is now established.
